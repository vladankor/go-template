# 📦 `pkg/`

## Назначение

Директория `pkg/` содержит **переиспользуемые библиотеки и утилиты**, которые:
- не зависят от бизнес-логики
- не привязаны к конкретному сервису
- могут быть использованы во внешних проектах

> Это место для **инструментальных компонентов**, не связанных с доменом: логгеров, хелперов, адаптеров и прочего.

---

## 🤝 Отличие от `internal/`

| Папка       | Импортируется извне? | Содержит бизнес-логику? |
|-------------|----------------------|--------------------------|
| `internal/` | ❌ Нет               | ✅ Да                    |
| `pkg/`      | ✅ Да                | ❌ Нет                   |

- Всё из `internal/` — **инкапсулировано**
- Всё из `pkg/` — **переиспользуемо**

---

## 📂 Примеры содержимого

pkg/ 
├── logger/ # Обёртка над zap/slog/logrus 
├── hash/ # Функции для хэширования паролей, токенов 
├── retry/ # Повторные попытки с backoff'ом 
├── configvalidator/# Библиотека валидации YAML-файлов конфигурации 
└── httpclient/ # Расширенный HTTP-клиент с логированием и ретраями

---

## 🔧 Требования к коду в `pkg/`

- 📦 **Чистая изоляция** — не импортируй `internal/*`
- ⚖️ **Минимальные зависимости** — не тащи весь мир
- 🧪 **Покрытие тестами** ≥ 90%
- 📚 **Документация через GoDoc** (комментарии на экспортируемые сущности)

---

## ✅ Best practices

| Рекомендация                             | Обоснование                                                   |
|------------------------------------------|---------------------------------------------------------------|
| Импортируй `pkg/*` через go module path  | Не через `../../pkg/logger`, а через `example.com/pkg/logger` |
| Избегай циклических зависимостей         | `pkg` не должен зависеть от `internal` или `cmd`              |
| Пиши как для open-source библиотеки      | Потому что рано или поздно ты её вынесешь туда :)             |
| Держи API стабильным                     | Переиспользуемый код не должен часто ломать интерфейсы        |

## 📌 Примечание

Если вы используете монорепу с несколькими микросервисами —  
`pkg/` может быть вынесен в отдельный модуль, например `github.com/your-org/libs`.
