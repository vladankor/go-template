# 🗃️ `internal/repo/`

## Назначение

`repo` (repository) — это **инфраструктурный слой**, реализующий контракты (`interfaces`) из слоя `usecase`.

> Здесь происходит работа с базой данных, кешами, поисковыми движками — всё, что связано с **постоянным хранением данных**.

---

## 🧱 Архитектурная роль

Слой `repo` реализует **паттерн репозиторий**, скрывая детали хранения и давая `usecase` и `domain` простой, абстрагированный интерфейс доступа к данным.

[ usecase ] ──► [ repo ] ──► [ DB / Cache / External store ]  
↑ │  
[ domain ]◄────┘ // интерфейсы, не зависящие от реализации


---

## 📦 Содержимое

| Компонент            | Назначение                                                                 |
|----------------------|----------------------------------------------------------------------------|
| `postgres/`          | Реализация репозиториев через `pgx` или `gorm`                             |
| `mongo/`             | Хранилище на MongoDB                                                       |
| `redis/`             | Репозитории для кэшей                                                      |
| `errors.go`          | Маппинг ошибок от СУБД к доменным                                           |

---

## 🔌 Пример: `UserRepository` на PostgreSQL

```go
package postgres

import "context"

type userRepo struct {
	db *pgxpool.Pool
}

func NewUserRepo(db *pgxpool.Pool) domain.UserRepository {
	return &userRepo{db: db}
}

func (r *userRepo) FindByID(ctx context.Context, id uuid.UUID) (*domain.User, error) {
	var user domain.User
	
	// Query хранить в отдельном файле (пример: repo/postgres/user/query.go)!
	err := r.db.QueryRow(ctx, `
        SELECT id, email, status, created_at
        FROM users WHERE id = $1
    `, id).Scan(&user.ID, &user.Email, &user.Status, &user.CreatedAt)

	if err == pgx.ErrNoRows {
		return nil, domain.ErrUserNotFound
	}

	return &user, err
}

```
