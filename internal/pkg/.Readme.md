# 🧰 `internal/pkg/`

## Назначение

`internal/pkg` содержит **внутренние вспомогательные библиотеки и утилиты**,  
которые используются в нескольких местах внутри микросервиса — но **не предназначены для переиспользования вне этого
сервиса**.

> Это своего рода "технический toolbox", не связанный с бизнес-логикой, но решающий _как_ что-то сделать — логгинг,
> ретраи, обёртки, валидация, хелперы и т.п.

---

## 🧱 Принципы

- В `internal/pkg` размещаются **общие, но внутренние** технические пакеты.
- Вне микросервиса к ним нельзя обращаться (благодаря `internal/`).
- Не помещайте сюда бизнес-логику — для неё есть `domain` и `usecase`.

---

## 🧩 Примеры содержимого

| Пример          | Описание                                                                 |
|-----------------|--------------------------------------------------------------------------|
| `marshaller/`   | Сериализаторы, адаптированные под внутренние форматы проекта             |
| `normalizer/`   | Нормализация специфичных данных (напр. форматирование номера клиента)    |
| `sqlbuilder/`   | Генерация SQL-запросов под конкретную бизнес-логику                      |
| `validator/`    | Валидация данных с бизнес-правилами, уникальными для проекта             |
| `rendering/`    | Отчёты, шаблоны, представления, специфичные для интерфейса проекта       |
| `featureflags/` | Обёртки над фич-флагами, работающие только с внутренней системой проекта |
| `worker/`       | Задачи выполняющиеся в заданную периодичность (cron-job)                 |

---

## 🚫 Что нельзя хранить в `internal/pkg`

| Путь          | Назначение                                                  |
|---------------|-------------------------------------------------------------|
| `logger/`     | Инициализация логгера, обёртки над `zap`, `zerolog` и пр.   |
| `retry/`      | Утилита для ретраев с backoff-политиками                    |
| `metrics/`    | Настройка и публикация метрик (`prometheus`, `otel`, и пр.) |
| `uuid/`       | Утилиты для генерации и валидации UUID                      |
| `errors/`     | Стандартизированная обёртка ошибок (`wrap`, `cause`, `is`)  |
| `httpclient/` | Настройки, middleware и обёртки для `http.Client`           |
| `ctxutil/`    | Расширения для работы с `context.Context`                   |

## 📦 Пример: `normalizer`

```go
package normalizer

import "strings"

func NormalizePhone(input string) string {
	input = strings.TrimSpace(input)
	input = strings.ReplaceAll(input, "+", "")
	input = strings.ReplaceAll(input, "(", "")
	input = strings.ReplaceAll(input, ")", "")
	input = strings.ReplaceAll(input, "-", "")
	input = strings.ReplaceAll(input, " ", "")

	if strings.HasPrefix(input, "8") {
		input = "7" + input[1:]
	}
	return input
}
```
