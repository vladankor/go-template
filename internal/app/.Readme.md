# ⚙️ `internal/app/`

## Назначение

`internal/app` — это **точка инициализации и запуска приложения**.  
Содержит функцию `Run()`, которую вызывает `main.go` из `cmd/`.

> Здесь соединяются все зависимости и конфигурации, а затем передаются в `api`, `usecase` и другие модули.

---

## 🧩 Основные обязанности

- Загрузка конфигурации
- Инициализация логирования, трейсинга, метрик и пр.
- Подключение к БД, Redis и другим внешним системам
- Сборка зависимостей (usecase, repo, API и т.д.)
- Запуск HTTP/GRPC-серверов, воркеров
- Обработка graceful shutdown

---

## 🗂 Возможная структура

| Путь         | Описание                                                           |
|--------------|--------------------------------------------------------------------|
| `builder.go` | Главный "конструктор", собирающий все зависимости.                 |
| `server.go`  | Запуск серверов, graceful shutdown, сигнал-обработка.              |
| `module.go`  | Wire-like DI: интерфейсы, провайдеры зависимостей.                 |
| `deps/`      | Модули инициализации (например, `db.go`, `logger.go`, `cache.go`). |
| `lifecycle/` | Логика запуска и остановки компонентов, хуки, готовность.          |

---

## ⚙️ Пример использования

```go
package app

func Run() {
	cfg := config.Load()
	log := logger.New(cfg.Env)
	db := database.Connect(cfg.DB)

	repos := repo.New(db)
	usecases := usecase.New(repos)
	handler := api.NewHandler(usecases)

	server := http.NewServer(cfg.HTTP.Port, handler)
	server.Start()
}
```
