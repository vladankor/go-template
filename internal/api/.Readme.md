# 🌐 `internal/api/`

## Назначение

Директория `api` реализует **входные интерфейсы** микросервиса: HTTP/REST, gRPC, WebSocket и другие точки взаимодействия
с внешним миром.  
Это уровень **транспортных адаптеров**, которые принимают запросы, валидируют, трансформируют и делегируют их
в `usecase`.

> Никакой бизнес-логики здесь быть не должно.

---

## 📌 Основные задачи

- Приём и обработка внешних запросов.
- Валидация входных данных.
- Преобразование DTO → domain-моделей и обратно.
- Вызов соответствующих usecase-интерфейсов.
- Формирование ответа в нужном формате (JSON, protobuf и т.д.).
- Обработка ошибок с маппингом на статус-коды и сообщения.

---

## 📂 Возможная структура

| Путь           | Описание                                                               |
|----------------|------------------------------------------------------------------------|
| `http/`        | HTTP-эндпоинты, роутинг, middleware, ответчики.                        |
| `grpc/`        | gRPC-сервер, реализации protobuf-интерфейсов.                          |
| `dto/`         | Структуры для входных/выходных данных, отдельные от `domain`.          |
| `middleware/`  | HTTP/gRPC middleware — логирование, аутентификация, трейсинг и др.     |
| `interface.go` | Контракты, описывающие поведение UseCase'ов (например: `UserUseCase`). |

---

## 🧭 Пример потока запроса

[HTTP-запрос]  
↓  
[router → handler]  
↓  
[DTO → валидация]  
↓  
[DTO → domain → usecase]  
↓  
[domain → DTO → JSON/gRPC-ответ]

---

## ✅ Пример кода (HTTP handler)

```go
package api

func (h *UserHandler) CreateUser(w http.ResponseWriter, r *http.Request) {
	var req dto.CreateUserRequest
	if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
		api.SendError(w, http.StatusBadRequest, "invalid request")
		return
	}

	user, err := h.userUseCase.Create(r.Context(), req.ToDomain())
	if err != nil {
		api.SendError(w, http.StatusInternalServerError, err.Error())
		return
	}

	api.SendJSON(w, dto.NewUserResponse(user))
}
```
