# 🧠 `internal/usecase/`

## Назначение

Слой `usecase` (или application logic layer) содержит **реализацию сценариев взаимодействия с системой** —  
всё, что отвечает на вопрос: **"Что делает ваш сервис?"**

> Это мост между `domain` (сущности и правила) и `repo` / `api` (хранилище и интерфейсы).

---

## 🧱 Архитектурная роль

`usecase` реализует **бизнес-сценарии**, опираясь на доменные интерфейсы и репозитории.  
Здесь **собирается в единое целое** вся логика обработки данных: от запроса до результата.

[ transport (api/grpc) ]  
↓  
[ usecase ]  
↙ ↘  
[ domain ] [ repo ]

---

## 🔧 Содержание

| Путь           | Описание                                                                |
|----------------|-------------------------------------------------------------------------|
| **User**       | UseCase-реализация операций с пользователями                            |
| **Auth**       | Логика аутентификации и авторизации                                     |
| **Order**      | Обработка заказов, бизнес-правила для них                               |
| **Interfaces** | Контракты, описывающие поведение хранилищ (например, `UserRepository`). |
| **Factory**    | Провайдер UseCase'ов с внедрением зависимостей (`NewXUseCase(...)`)     |

---

## 💼 Пример `user/`

```go
package user

import (
	"context"
	"fmt"
)

type UserUseCase struct {
	repo   domain.UserRepository
	hasher PasswordHasher
	logger Logger
}

func NewUserUseCase(r domain.UserRepository, h PasswordHasher, l Logger) *UserUseCase {
	return &UserUseCase{repo: r, hasher: h, logger: l}
}

func (u *UserUseCase) Register(ctx context.Context, dto RegisterDTO) (*UserDTO, error) {
	hash, err := u.hasher.Hash(dto.Password)
	if err != nil {
		return nil, fmt.Errorf("hashing error: %w", err)
	}

	user := &domain.User{
		ID:       uuid.New(),
		Email:    dto.Email,
		Password: hash,
	}

	err = u.repo.Save(ctx, user)
	if err != nil {
		return nil, err
	}

	u.logger.Infof("new user registered: %s", user.ID)

	return ToUserDTO(user), nil
}

```
