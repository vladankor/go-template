# 🧪 `integration-test/`

## Назначение

`integration-test` содержит **интеграционные тесты**, проверяющие корректность взаимодействия **всех слоёв приложения** между собой и/или с внешними зависимостями (БД, Kafka, Redis и др.).

> Это не юнит-тесты. Здесь проверяется **поведение системы целиком**, а не изолированных функций.

---

## 🔍 Когда запускать

Интеграционные тесты обычно запускаются:
- **локально**, при ручной проверке фич
- **в CI**, после `build` и `migrate`
- **в pre-deploy pipeline** на stage-среде

## ⚙️ Как устроено

Каждый тест:
- запускает миграции (`migrations/`)
- прогоняет SQL-фикстуры (`testdata/`)
- вызывает реальные методы `usecase`, `repo`, `api`
- валидирует **конечный результат**

Обычно подключение к базе происходит через `docker-compose`, `TestMain` или `setup.go`.

---

## 🧱 Пример

```go
package integration_test

func TestUserRegistration(t *testing.T) {
    ctx := context.Background()

    // Arrange
    db := testdb.ConnectAndMigrate()
    repo := postgres.NewUserRepo(db)
    uc := usecase.NewUserUseCase(repo, hash.New(), logger.Noop())

    // Act
    user, err := uc.Register(ctx, RegisterDTO{Email: "test@example.com", Password: "123"})

    // Assert
    require.NoError(t, err)
    require.Equal(t, "test@example.com", user.Email)
}
```
