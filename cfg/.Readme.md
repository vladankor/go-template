# 📁 `config/`

## Назначение

Директория `config` содержит **описание конфигурационных схем** и вспомогательные структуры для загрузки, валидации и хранения конфигурационных параметров микросервиса.  
Эти параметры, как правило, поступают из внешних YAML/ENV-файлов, переменных окружения или систем управления конфигурацией.

---

## 📌 Основные задачи

- Декларативное описание конфигурационных структур (например, `AppConfig`, `DBConfig`, `HTTPConfig`).
- Поддержка различных окружений (`local`, `dev`, `prod`) через внешние конфигурационные файлы.
- Валидация конфигураций на старте приложения.
- Обеспечение централизованного доступа к параметрам конфигурации.

---

## 📂 Содержимое

| Файл/Папка          | Описание                                                                                                                       |
|---------------------|--------------------------------------------------------------------------------------------------------------------------------|
| `config.go`         | Основная структура конфигурации, методы загрузки и инициализации.                                                              |
| `loader.go`         | Утилиты для чтения и парсинга YAML/ENV файлов (Только при отсутствии библиотеки для чтения конфигурации)                       |
| `validator.go`      | Функции для проверки корректности значений и наличия обязательных полей. (Только при отсутствии библиотеки для чтения конфига) |

---
